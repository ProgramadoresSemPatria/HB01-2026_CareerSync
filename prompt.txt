CONTEXTO DO PROJETO
Tenho um back-end em Python com FastAPI para uma aplicação de análise de currículo vs. vaga de emprego. Já existe uma feature funcional num endpoint /analyze que recebe o currículo (PDF), título e descrição da vaga, chama uma LLM, e retorna o resultado. Os dados atualmente ficam no localStorage do front-end.
Preciso refatorar o back-end para:

Persistir os dados num banco de dados
Separar o endpoint monolítico /analyze nos novos endpoints do contrato
Implementar cache no banco: consultar a LLM apenas se o dado ainda não existir

PADRÃO DE IMPLEMENTAÇÃO (helper reutilizável)
Para evitar repetição nos endpoints com cache, extraia uma função utilitária com a seguinte lógica:
pythonasync def get_or_generate(analysis_id, field_name, generate_fn):
    analysis = db.get(analysis_id)  # busca no banco
    if not analysis:
        raise HTTPException(404)
    
    cached = getattr(analysis, field_name)
    if cached is not None:
        return cached
    
    result = await generate_fn(analysis)  # chama a LLM
    db.update(analysis_id, {field_name: result})  # salva no banco
    return result
